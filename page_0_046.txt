La tache décrit le résultat

Le texte sur le Post-It compte. Passer quelques secondes de plus pour décrire le travail correspondant au Post-It peut faire une différenfe importante.

Ne marquez pas simplement un mot clé ou  une vague description du travail - décrivez l'état dans lequel doit se trouver le système la tache réalisée. Une autre manière de le dire: Ne décrivez pas le travail, décrivez le résultat.

Quand vous appliquez cette règle, le but de la tache et ce qui est effectivement demandé deviennent beaucoup plus clair pour l'équipe.

Task describes result

The texts on the Post-its matter. Spending a couple of extra seconds to describe the work a Post-it refers to can make a huge difference.

Don’t simply write a keyword or a vague description of the work – describe what state we are in when the task has been completed. Another way to phrase the same thing: don’t describe the work, describe the result.

When applying this policy, it becomes much clearer for the team what the goal of the task is and what’s expected.
